# Sūtra

**Rules that hold everything together.**

> *सूत्र (sūtra)* — Sanskrit. Literally: thread or string.
> By extension: a rule, a guiding principle, an aphorism so precise it needs no elaboration.

---

## What is Sūtra?

In Sanskrit, a *sūtra* is a thread — but not an ordinary one. It is the minimal, essential rule that holds a larger system together. Buddhist and Hindu philosophical texts are called sūtras because each line is a compressed truth: a single thread woven into a tapestry of meaning.

A project is the same thing.

Not a list. Not a pile of tickets. A network of threads — tasks that pull on each other, deadlines that ripple, people whose work depends on someone else's. **Sūtra is where those threads live.**

---

## What does it do?

Sūtra is a **GraphQL API** for task and project management — built for teams who want structure without ceremony.

- Organize work into **projects** and **tasks**
- Assign tasks to team members, set priorities and due dates
- Track status from `TODO` → `IN_PROGRESS` → `DONE`
- Query exactly what you need — nothing more, nothing less
- Authenticate with JWT, own your data end to end

One endpoint. One schema. Every thread, accounted for.

## The name

Most productivity tools are named after motion — *Flow*, *Sprint*, *Stride*, *Dash*.
Sūtra is named after **structure**. The invisible thread that keeps things from unraveling.

When a project is running well, you don't see the thread. You only notice it when it breaks.
